# Stable Diffusion Forge Setup Guide

## Installation

### 1. Clone Repository

```bash
cd $HOME/dev
git clone https://github.com/lllyasviel/stable-diffusion-webui-forge.git
cd stable-diffusion-webui-forge
```

### 2. Create Pixi Environment

### Environment Activation

Activate the `sd-forge` environment:

```bash
conda activate sd-forge
```

### Dependency Installation

Install the required packages and specific versions for CUDA 12.4:

```bash
pip install -r requirements.txt
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu124
pip install xformers --index-url https://download.pytorch.org/whl/cu124
pip install "numpy<2"
```

## Execution

### Launch with Web UI

Start the application with performance optimizations:

```bash
python launch.py --cuda-malloc --xformers --pin-shared-memory --always-high-vram
```

### Launch API Only

Start the application in headless API mode:

```bash
python launch.py --api --nowebui
```

## Model Installation

### Download Stable Diffusion Models

Models should be placed in the `models/Stable-diffusion/` directory.

#### Example: Juggernaut-XL-v9

Download from Hugging Face:

```bash
cd models/Stable-diffusion
wget https://huggingface.co/RunDiffusion/Juggernaut-XL-v9/resolve/main/Juggernaut-XL_v9_RunDiffusionPhoto_v2.safetensors
```

Alternative: Download using huggingface-cli:

```bash
cd models/Stable-diffusion
huggingface-cli download RunDiffusion/Juggernaut-XL-v9 \
  Juggernaut-XL_v9_RunDiffusionPhoto_v2.safetensors \
  --local-dir . \
  --local-dir-use-symlinks False
```

#### Verify Installation

Check that the model file is present:

```bash
ls -lh models/Stable-diffusion/
```

The safetensors file should be approximately 6-7 GB in size.

## Troubleshooting

### Missing Dependencies

If you encounter errors about missing Python packages (e.g., `joblib`, `pycairo`, `svglib`), add them using pixi:

#### Add Python packages via pixi:

```bash
# Add missing Python packages
pixi add --pypi joblib pycairo svglib
```

#### Add system dependencies:

```bash
# Add cairo and pkg-config for pycairo support
pixi add cairo pkg-config
```

#### Apply changes:

```bash
pixi install
```

### Common Errors

- **`ModuleNotFoundError: No module named 'joblib'`**: Run `pixi add --pypi joblib`
- **`Dependency "cairo" not found`**: Run `pixi add cairo pkg-config`
- **`Failed to install svglib`**: Run `pixi add --pypi svglib pycairo`

After adding dependencies, always run `pixi install` to update your environment.

## gen_train_data_v2.py

This script generates a dataset for training a model using the Stable Diffusion Web UI API. It uses a base prompt and variations to generate a set of images, which are then saved to a specified directory.

### Usage

```python
python gen_train_data_v2.py
```

### Configuration

The script has a number of configurable parameters, including:

- `PORT`: The port number on which the Stable Diffusion Web UI API is running.
- `URL`: The URL of the Stable Diffusion Web UI API.
- `SAVE_DIR`: The directory in which the generated images will be saved.
- `BASE_PROMPT`: The base prompt used to generate the images.
- `VARIATIONS`: A list of variations used to generate the images.

### Example

```python
python gen_train_data_v2.py
```

### Notes

The script uses the `requests` library to make HTTP requests to the Stable Diffusion Web UI API. It also uses the `base64` library to decode the images generated by the API.
